## Context

The course already has 8 PowerPoint slide decks (`slides/Week01_Startup.pptx` … `Week08_Startup.pptx`) generated by `create_slides.py`. The website makes that content accessible to students online without requiring a login, and lets them download each week's slides directly.

---

## Goals

- Public, no-login website displaying weekly course content
- Thai-primary language (English terms stay in brackets/sub-labels as in the slides)
- Light-mode design matching the slide color palette
- One download button per week for the `.pptx` file

---

## Tech Stack

| Layer | Choice |
|---|---|
| Framework | Vue 3 (Composition API) + Vite |
| Styling | CSS custom properties (no extra framework needed) |
| Data | Static JS file (`src/data/weeks.js`) |
| Hosting | Static file host (Vercel / GitHub Pages / local) |

No router — single SPA page with anchor-scroll navigation.

## Page Layout

```
NAVBAR  (sticky top)
  Logo / ชื่อรายวิชา  |  Anchor links: W1 W2 W3 … W8

HERO SECTION
  ชื่อรายวิชาภาษาไทย + อังกฤษ
  ชื่ออาจารย์ / คณะ / มหาวิทยาลัย
  จำนวนหน่วยกิต / จำนวนสัปดาห์

WEEK 1 SECTION  (id="week-1")
  ┌ WeekHeader: "สัปดาห์ที่ 1" + ชื่อหัวข้อ TH + EN + CLO badges
  ├ SlideCards: grid ของ Card แต่ละ slide (title + bullet list)
  ├ ActivityCards: 3 columns — บรรยาย / เวิร์กชอป / กิจกรรมที่ 3
  ├ TakeawayList: รายการ ✓ 4 ข้อ
  └ DownloadBtn: "⬇ ดาวน์โหลดสไลด์ Week 1 (.pptx)"

WEEK 2–8 SECTIONS  (เหมือนกัน)

FOOTER
  ข้อมูลรายวิชา / อาจารย์ / มหาวิทยาลัย
```

## Component Tree

```
App.vue
├── NavBar.vue          sticky nav with 8 anchor links
├── HeroSection.vue     course banner
├── WeekSection.vue     (used ×8, receives week data as prop)
│   ├── WeekHeader      number badge + TH/EN title + CLO/method/media chips
│   ├── SlideCards      grid — one card per slide topic
│   ├── ActivityCards   3-column activity grid
│   ├── TakeawayList    checkmark list
│   └── DownloadBtn     <a :href> link to /slides/WeekNN_Startup.pptx
└── FooterBar.vue
```

---

## Data Layer

`src/data/weeks.js` — direct JS port of the `weeks` list in `create_slides.py`. Each object:

```js
{
  week: 1,
  title_th: "...",
  title_en: "...",
  clos: "...",
  method: "...",
  media: "...",
  slides: [["title", ["bullet", ...], "note?"], ...],
  activities: [["title", "body"], ...],
  takeaways: ["...", ...],
  next_week: "...",
}
```

Slide files live at `public/slides/Week01_Startup.pptx` … `Week08_Startup.pptx` so Vite serves them as static assets.
